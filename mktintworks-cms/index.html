Project Overview

Custom CMS foundation for a two-repository Cloudflare architecture.

This bootstrap captures the non-negotiable parts of PRD Section 1: the module inventory, the public website versus CMS split, the Cloudflare stack, the editable page surface map, and the seed product catalog.

Bootstrap Only

No auth, API, D1, R2, KV, or Access wiring has been added yet.

10 Seed products

4 Current-state gaps frozen

Three Layers

Architecture

Layer 1

Public Website

Static HTML/CSS/JS. Reads structured content at build time and uses real-time reads only where PRD requires it.

Layer 2

CMS Admin Panel

Separate application at `admin.mktintworks.com`, visually aligned with the website and protected by Cloudflare Access.

Layer 3

Worker API

Cloudflare Worker for auth, D1 writes, R2 uploads, Workers AI calls, PDF generation, analytics, and deploy hooks.

Repository Split

Website and CMS stay independent.

Repo 1

mktintworks-website

Public site files, Pages deployment, rebuild-on-content-save workflow.

Repo 2

mktintworks-cms

Admin interface and CMS-side logic, deployed separately so CMS code changes never redeploy the public site.

Core Principles

Section 1 rules that cannot drift later.

A child could use it

Plain-English labels, obvious actions, zero hidden complexity.

Premium feel

Same colors, fonts, and visual quality as the MK Tintworks website.

Zero monthly cost

Cloudflare free-tier-first stack with no paid SaaS dependencies.

Fast

Website updates target the 20-30 second post-save window defined in the PRD.

Secure

Cloudflare Access at the edge plus API-level JWT checks on every protected endpoint.

Mobile-capable

Admin layout must remain usable on a phone, not just on a laptop.
